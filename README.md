# card-game
仅用于执行课程教学内容练习/零基础新手
本项目架构采取内部底层内存管理隔离总管上层资源，由于没有对上下文进行封装，因此所有事件分发和交互内容做成接口形式和模块放在上层interaction中
由main部分调度整合模块


v1.0版本
实现对于原型游戏可行性检测，包括但不限于游戏玩法，策划内容规划，同时包括对于项目架构的初步构想设计
实现最简化可运行程序，实现控制台字符交互，和内部底层数据交互


v2.0版本
初步搭建框架，实现数据逻辑封层隔离，对于底层工具与上层交互层分离，对模块职责做明确职责划分，对成员负责模块明确分工范围
实现简易基础通用数据结构，辅助后续开发全局唯一资源管理以及对于底层工具的设计

分工：由组员林负责底层资源管理模块，提供统一资源协议接口，同时负责团队规范代码内容、形式和项目架构；
由组员杰负责输出图形模块，即视觉交互效果；
由组员樾负责音频模块，实现输出音频；
由组员浩负责事件系统、游戏状态机，整合数据config与逻辑logic模块内容；
所有组员协助完成中间层内容，除开各自负责专职模块外，数据结构与算法基本存放于tool层，由所有组员学习、使用、共同完成，模块通用思想，方法同理。

v3.0版本
暂定：完成中间层logic与config搭建，所有队员辅助完成，同时搭建基础交互功能：音频，图形输出，事件输出等等
